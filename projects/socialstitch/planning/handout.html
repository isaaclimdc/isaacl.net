Due Sat 6-Oct 11:59pm, early-bird bonus Thu 4-Oct 11:59pm.

Read these instructions first!  They are similar to hw1, but differ in important ways.  Read carefully!

This is an open-ended problem to write the client-side of a web app that showcases the major concepts we have covered so far in this course.  The rules are:

  • You must not write a game.  Anything else is fine, though we recommend something of lasting value (such as, say, an interactive online portfolio that you might provide to prospective employers).
  • Ideally, you will complete this by Thu 4-Oct, but we have extended the deadline to Saturday, though that will shorten your time for hw5.  And so you will earn a 5-point early-bird bonus for submitting by Thursday 4-Oct at 11:59pm.
  • You must showcase the major concepts we have covered in weeks 1-4 and then in a natural and compelling way.  If your project does not easily allow for this, change your project accordingly, since it is required.  That said, you can pick where the focus of your project will be.  So if a Canvas does not naturally fit in your original design, don't throw out your design, just add a smart little Canvas object somewhere while preserving the overall design and intent.
  • You should expect to spend 10-15 hours each on this project.  Some students invested far more than that on hw1 (our last open-ended project).  You may do so, but we do not expect that, and it is not required, as we expect 10-15 hours to be just fine to accomplish everything required to obtain a solid A on this project.
  • Projects should show a level of sophistication in keeping with 10-15 hours of effort given how much you should already know after completing hw1-3.  They should be neither too simplistic nor too complex.  They should also be interesting, though we will take a broad view of this.
  • Your code does not have to run on mobile devices, but it must run on both Chrome and Firefox on a desktop. Be sure to test it on both.  That said, if you would like to use an API that is only available in one of Firefox or Chrome, make sure your web app still shows some useful information in a nice, user-friendly way on the other browser (for example, maybe it informs users that they will get the full experience only on browser X).
  • We encourage you to work in 2-to-3-person groups, but you may request to work alone if you wish.  All such requests should be made by email to the instructor no later than Sunday 23-Sep.
  • Each group will make a single submission. Do not just split the work among you and merge it at the end. You are also responsible for thoroughly understanding every single line of code your group submits.
  • You may not use any server-side computation for this project, except for publicly-available web services that you did not write.  That is, you need to write an AJAX client, but only consuming pre-existing services.  Otherwise, your project must work when served by a server that only supports static HTML pages (that is, again, no server-side computation).
  • Besides this server-side restriction, you may use any 3rd-party libraries or tools that you wish.  That said, there must not be any installation procedure of any kind for your project.  To grade your project, we will load the top-level HTML file (preferably index.html), and that's it.
  • You must create an entirely new piece of work, and not reuse anything you have written before this course.  You may extend work that you submitted earlier in this course, but you will only be graded on the new contributions, which must be clearly marked.
  • Have fun. Truly!!!

How will you be graded?

The grading is naturally a bit subjective on such an open-ended and creative project, but we try very hard to be reasonable and predictable and to maintain consistency and fairness across all submissions.  Here is an outline of the rubric:

  1. Required Elements [35 pts; 5 pts each]
    Your project must demonstrate clear and effective use of each of the following major concepts we have covered:
    1. Javascript (including creating your own objects with inheritance)
    2. Canvas (somehow this must be included meaningfully)
    3. HTML (including a reasonably wide range of techniques, such as tables, forms with form validation, etc)
    4. CSS (including a reasonably range of techniques, such as reset, pseudo-selectors, fixed and fluid layout, transitions and animations, etc)
    5. DOM manipulation
    6. AJAX and/or JSON-P client
    7. jQuery
       
  2. Robust App [20 pts]
    Your web app should work.  It should load and run with no exceptions, and clearly perform some easily-understood task, and it should perform it properly under normal usage.  It also should not crash or hang or otherwise enter some error state, even in the face of undesired input.
          
  3. User Interface Design [20 pts]
    You should have a reasonably polished user interface with a clean, pleasing visual design, and with intuitive controls and navigation.
          
  4. Effort [20 pts]
    Once again:  you are expected to invest at least 10-15 hours into this project, and it is further expected that your project will clearly reflect the quantity and quality of design and craftsmanship that goes along with that much time on this task.
     
  5. Code Design / Style [5 pts]
    Your project should be divided properly into HTML, CSS, and JS, with clear designs in each.  Your code should abide by all the style guidelines we have discussed in this course.
       
  6. Bonus [0 to 10+ pts]
    For extraordinary work, we may award bonus points.   Please discuss more ambitious bonus attempts with the TA's or the instructor prior to investing too much time into them!
